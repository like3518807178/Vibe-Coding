# AGENTS.md

你是这个仓库中的代码代理。请严格遵守以下规则。

## 1. 项目总目标
本项目是一个 Linux + C++17 的 TinyIM 迭代项目。
版本从 V0 到 V8 逐步推进。
当前只允许完成 TASK.md 中定义的版本目标，不允许越级开发。

## 2. 通用开发规则
1. 使用 C++17。
2. 优先小步修改，不做与当前版本无关的大重构。
3. 每次改动后，必须保证代码可编译、可运行。
4. 不要一次实现多个版本。
5. 不要提前引入后续版本的特性。
6. 如果需求不明确，优先依据：
   - TASK.md
   - docs/roadmap_v0_v8.md
7. 代码风格保持简洁，优先标准库，不引入重量级框架。
8. 错误处理必须明确，不能静默吞错。
9. 保持目录结构清晰，命名直白。
10. README 需要随着版本推进同步更新。

## 3. 工程环境
- 工作目录：/home/like/VibeCoding
- 平台：Ubuntu (VMware)
- 编辑器：VSCode + Codex 扩展
- 编译工具：CMake + g++
- 目标：做出可逐版验收、可讲解、可写简历的项目

## 4. 输出要求
每次开始任务前，先输出：
1. 任务理解
2. 实现计划
3. 将要修改的文件列表

完成代码后，必须输出：
1. 修改了哪些文件
2. 为什么这么改
3. 编译命令
4. 运行命令
5. 验收步骤
6. 当前仍未完成的内容

## 5. 严格限制
1. 不允许跳过当前版本去实现后续版本内容。
2. 不允许为了“以后扩展”而提前堆复杂抽象。
3. 优先交付当前版本的最小可运行实现。
4. 没有明确要求时，不自动加入线程池、数据库、epoll、JSON、配置系统、日志系统等后续模块。
